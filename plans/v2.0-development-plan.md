# VCM v2.0 开发计划

> 分析时间：2026-03-14
> 分析视角：忠实用户、高频用户、商业分析师

---

## 一、忠实用户视角（深度使用体验）

### 1. 使用统计与成本追踪 ⭐⭐⭐

```bash
vcm stats              # 显示各工具使用频率、API 调用估算
vcm cost               # 月度成本报告（基于公开定价估算）
vcm quota              # 查看各工具配额剩余
```

- **痛点**：用户同时使用多个工具，但不清楚哪个用得多、成本如何
- **价值**：帮助用户优化工具选择，控制开支

### 2. 工具对比功能 ⭐⭐⭐

```bash
vcm compare claude-code gemini-cli aider
```

- **痛点**：选择工具时缺少横向对比，需要查多个网站
- **价值**：一次展示功能、价格、模型能力对比表

### 3. 使用历史与收藏 ⭐⭐

```bash
vcm history            # 最近使用的工具和命令
vcm favorite           # 收藏的工具列表
vcm favorite add claude-code
```

- **痛点**：工具多了容易忘记哪个好用
- **价值**：快速访问常用工具

## 二、高频用户视角（效率与便捷）

### 4. 别名与快捷启动 ⭐⭐⭐

```bash
vcm alias cc claude-code    # 设置别名
vcm cc                      # 直接启动
vcm .                       # 启动默认工具（类似 npm run）
```

- **痛点**：每天输入完整命令太繁琐
- **价值**：大幅提升日常使用效率

**实现方案**：
- 配置文件存储别名映射
- 支持命令行快捷调用
- 自动补全支持别名

### 5. 项目级配置 ⭐⭐⭐

```bash
vcm project init            # 当前项目绑定工具配置
vcm project use claude-code --model claude-sonnet-4
vcm project status          # 显示当前项目配置
```

- **痛点**：不同项目需要不同工具配置（如 API Key、模型选择）
- **价值**：自动切换上下文，避免配置冲突

**实现方案**：
- 项目根目录创建 `.vcm/config.toml`
- 自动检测当前项目配置
- 支持环境变量覆盖

### 6. 工具链/预设管理 ⭐⭐

```bash
vcm preset create frontend    # 创建工具组：aider + cursor + copilot
vcm preset use frontend       # 一键激活一组工具
vcm preset list               # 列出所有预设
```

- **痛点**：特定场景需要多工具配合
- **价值**：一键切换工作模式

### 7. 后台服务管理 ⭐⭐

```bash
vcm service start ollama     # 启动本地模型服务
vcm service status           # 查看运行状态
vcm service logs             # 查看日志
vcm service stop ollama      # 停止服务
```

- **痛点**：Ollama 等本地服务需要单独管理
- **价值**：统一管理所有 AI 服务

## 四、隐藏价值功能（用户痛点但不愿公开讨论）

> 这部分功能触及商业利益灰色地带，但恰恰是用户最真实的需求

### 12. 免费额度聚合器 ⭐⭐⭐💰

```bash
vcm free --aggregate          # 显示所有可用免费额度总和
vcm free --optimize           # 推荐最优免费组合策略
vcm free --rotate             # 在多个免费工具间轮换使用
```

- **隐含需求**：用户想用最强模型但不想付费
- **真实场景**：Gemini 免费100次/天 + Kiro 50次/月 + Kilo $25额度 = 大量免费调用
- **为什么不说**：像"薅羊毛"，不太光彩
- **商业价值**：**极高** - 用户增长利器

**实现方案**：
```bash
$ vcm free --aggregate

🎁 免费额度聚合面板

工具            免费额度              专业级模型        状态
─────────────────────────────────────────────────────────
Gemini CLI      100 次/天            ✓ Gemini 2.5 Pro   可用
Kiro            50 次/月             ✓ Claude 4 Sonnet  可用
Kilo Code       $25 额度 (一次性)     ✓ Claude Opus 4.5  可用
Ollama          无限 (本地)          ✓ Qwen2.5-Coder    可用
OpenCode        无限 (BYOK)          ✓ 多模型支持       可用

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
聚合潜力: ~150+ 次/月 专业级模型调用 (无需付费)
```

### 13. 多账号/多 Key 管理 ⭐⭐⭐🔑

```bash
vcm key add claude-personal xxx      # 个人账号
vcm key add claude-work yyy          # 工作账号
vcm key list                         # 列出所有 Key
vcm key switch claude-work           # 切换账号
vcm key rotate                       # 轮换使用（规避单账号限制）
```

- **隐含需求**：多账号、多区域、规避单账号限制
- **真实场景**：开发者有多个 Anthropic 账号，或公司账号+个人账号
- **为什么不说**：可能违反服务条款
- **商业价值**：**高** - 高级用户刚需

**实现方案**：
```toml
# ~/.config/vcm/keys.toml (加密存储)
[claude-code.keys]
personal = { key = "sk-ant-xxx...", active = true }
work = { key = "sk-ant-yyy...", active = false }
trial = { key = "sk-ant-zzz...", expires = "2024-03-20" }

[claude-code.settings]
rotation_enabled = true
rotation_interval = "hourly"  # hourly/daily/per-request
```

### 14. 配额追踪与超限保护 ⭐⭐⭐🎯

```bash
vcm quota watch                 # 实时监控用量
vcm quota status                # 各工具使用量统计
vcm quota warn 80%              # 设置警告阈值
vcm quota limit --hard 100      # 硬限制（超限自动停止）
```

- **隐含需求**：精确控制用量，避免超额扣费
- **痛点**：不知道用了多少，月底账单吓人
- **为什么不说**：显得"斤斤计较"
- **商业价值**：**高** - 防止超额扣费

**实现方案**：
```bash
$ vcm quota watch

📊 配额监控面板 (实时)

工具            已用      限额      剩余      状态
───────────────────────────────────────────────────
Claude Code     45 次     无限      -         💳 按量计费
Gemini CLI      78 次     100/天    22 次     ⚠️ 即将达到限额
Kiro            12 次     50/月     38 次     ✓ 正常
Ollama          234 次    无限      -         ✓ 本地无限

⚠️ 警告: Gemini CLI 今日剩余 22 次，建议切换备用工具
```

### 15. 智能降级/自动切换 ⭐⭐⭐⚡

```bash
vcm fallback setup              # 配置降级策略
vcm fallback add claude-code -> gemini-cli -> ollama
vcm fallback status             # 当前降级链状态
```

- **隐含需求**：不间断工作，无缝切换
- **真实场景**：Claude 用完了自动切 Gemini，Gemini 用完切本地
- **为什么不说**：不想绑定单一厂商
- **商业价值**：**极高** - 用户粘性

**实现方案**：
```bash
$ vcm fallback setup

配置降级链:
主力工具        备选工具           兜底工具
───────────────────────────────────────────────
claude-code → gemini-cli → ollama (qwen2.5-coder)

触发条件:
[x] 主力工具配额用尽
[x] 主力工具响应超时 (>30s)
[x] 主力工具 API 错误
[ ] 主力工具成本超限

是否启用智能降级? [Y/n]
```

### 16. 价格监控与优惠提醒 💸

```bash
vcm price-watch                 # 监控价格变化
vcm price-history claude-code   # 价格历史
vcm price-alert --drop 20%      # 降价提醒
vcm deal                        # 当前最优优惠
```

- **隐含需求**：等降价、省钱
- **真实场景**：Anthropic 降价、新工具首发优惠
- **为什么不说**：显得"贪便宜"
- **商业价值**：**高** - 差异化功能

### 17. 本地模型一键部署 ⭐⭐🏠

```bash
vcm local install qwen2.5-coder:32b   # 安装本地模型
vcm local list                        # 已安装模型
vcm local benchmark                   # 性能测试
vcm local vs cloud                    # 本地 vs 云端对比
```

- **隐含需求**：完全免费、隐私保护、无网络依赖
- **真实场景**：敏感项目不能上传云端
- **为什么不说**：显得对云端工具"不忠诚"
- **商业价值**：**高** - 离线场景刚需

### 18. "影子"工具策略 👥

```bash
vcm shadow add claude-pro       # 付费主力（明面）
vcm shadow add gemini-free      # 免费备用（暗地）
vcm shadow add ollama-local     # 本地兜底（终极备份）
vcm shadow status               # 影子配置状态
```

- **隐含需求**：明面上用一个，背地里有一堆备胎
- **真实场景**：告诉同事"我用 Claude"，实际偷偷用 Gemini 免费
- **为什么不说**：显得"不专一"

### 19. 试用追踪与自动提醒 ⭐⭐

```bash
vcm trial track claude-pro --days 7    # 标记试用期
vcm trial list                         # 所有试用期状态
vcm trial remind --before 3d           # 提前3天提醒
vcm trial cancel-all                   # 一键取消所有试用
```

- **隐含需求**：试用多个工具，追踪试用期，避免意外扣费
- **真实场景**：注册了10个试用，忘记取消被扣款
- **为什么不说**：只想白嫖试用

## 六、v2.0 版本规划

### Phase 1: P0 功能（核心体验提升 + 隐藏价值）

**目标**：提升日常使用效率，解决用户真实痛点

1. **别名系统**
   - 配置文件：`~/.config/vcm/config.toml`
   - 新增命令：`vcm alias <alias> <tool>`
   - 支持快捷调用：`vcm <alias>`
   - 自动补全支持

2. **工具对比**
   - 新增命令：`vcm compare <tool1> <tool2> [tool3...]`
   - 对比维度：模型能力、价格、免费额度、安装方式
   - 表格输出，支持 JSON 格式

3. **免费额度聚合器** 🆕
   - 新增命令：`vcm free --aggregate`
   - 聚合所有工具的免费额度
   - 推荐最优免费组合策略
   - 显示专业级模型免费获取途径

4. **配额追踪保护** 🆕
   - 新增命令：`vcm quota watch/status/warn`
   - 实时监控各工具使用量
   - 超限警告和硬限制
   - 成本估算

### Phase 2: P1 功能（深度使用体验）

**目标**：帮助用户更好地管理和优化工具使用

1. **使用统计**

   - 估算 API 成本
   - 导出报告

2. **项目级配置**
   - `.vcm/config.toml` 配置文件
   - 自动检测项目配置
   - 环境变量覆盖
   - 项目间配置隔离

3. **智能降级切换** 🆕
   - 新增命令：`vcm fallback setup/add/status`
   - 配置主力→备选→兜底降级链
   - 自动切换触发条件配置
   - 无缝切换不中断工作

4. **多账号管理** 🆕

   - 支持账号切换和轮换

## 八、风险与挑战

### 技术风险

1. **使用统计准确性**：需要可靠的方式追踪工具调用
2. **成本估算误差**：API 定价变化、不同模型价格差异
3. **项目配置优先级**：全局配置 vs 项目配置的冲突处理
4. **跨平台兼容性**：Windows/macOS/Linux 的路径差异

### 隐藏功能相关风险 ⚠️

1. **多账号管理合规性**
   - 风险：可能违反部分厂商服务条款
   - 缓解：明确告知用户风险，提供免责声明
   - 备选：仅作为"环境管理"功能，不主动鼓励轮换

2. **免费额度聚合争议**
   - 风险：厂商可能认为这是"薅羊毛"工具
   - 缓解：强调这是帮助用户"了解自己的权益"
   - 备选：功能命名改为"免费额度概览"，更中性

3. **API Key 存储安全**
   - 风险：多个 Key 集中存储，安全要求高
   - 缓解：使用系统级加密存储（keyring/keychain）
   - 备选：不存储完整 Key，仅存储引用

4. **配额追踪准确性**
   - 风险：各厂商 API 不一定提供用量查询
   - 缓解：基于本地使用记录估算
   - 备选：提供手动校准功能

### 声誉风险

- 部分功能可能被厂商视为"灰色工具"
- 建议在文档中明确：
  - "本工具仅帮助用户管理自己的账号和配额"
  - "请遵守各厂商服务条款"

## 十、隐藏功能的商业策略思考

### 为什么这些功能有价值但没人做？

1. **厂商利益冲突**：帮用户省钱 = 厂商少赚钱
2. **灰色地带**：多账号、配额最大化可能违规
3. **用户心理**：不想承认自己"穷"或"薅羊毛"

### VCM 的定位策略

- **对外**：工具管理器，帮助用户"了解和配置"工具
- **对内**：成本优化器，帮助用户"聪明地使用"工具
- **本质**：站在用户这边，做用户的"AI 工具财务顾问"

### 可能的变现路径

| 路径 | 说明 | 适用阶段 |
|------|------|---------|
| 免费版 | 基础功能 + 隐藏功能 | 用户增长期 |
| Pro版 | 高级统计 + 云同步 + 团队 | v2.2 |
| 企业版 | 合规审计 + 私有部署 | v3.0 |
| 数据服务 | 匿名使用数据报告 | v2.5+ |
